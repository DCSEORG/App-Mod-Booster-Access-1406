Swinton Go! Insurance - New App Limitations
==========================================
Generated: 2026

1. REAL-TIME DVLA VEHICLE LOOKUP
   The vehicle registration lookup does NOT connect to the actual DVLA API.
   A stubbed timeout (1.1s) simulates the lookup and falls back to a hard-coded
   Volkswagen Golf record. Only the registration "D500 NLE" returns the stored
   demo vehicle; all other registrations also return the same Golf fallback.
   To enable real DVLA lookups, integrate with the DVLA Vehicle Enquiry Service API
   (https://developer-portal.driver-vehicle-licensing.api.gov.uk/).

2. PHONE NUMBER PLACEHOLDER
   The contact phone number is shown as "0800 XXX XXXX". The real Swinton Go!
   customer service number must be confirmed and updated before going live.

3. DUMMY PRICING ENGINE
   The quote calculation uses a simplified actuarial model for demonstration
   purposes only. Real production pricing would be driven by an insurer panel
   rating engine / aggregator API. The "Buy now" and "Pay annually" buttons
   show an alert confirming this is a demo and do not connect to any payment system.

4. NO BACKEND / PERSISTENCE
   All form data is stored exclusively in sessionStorage (browser tab memory).
   Data is lost when the tab is closed. There is no server-side storage, user
   accounts, quote retrieval, or email confirmation functionality.

5. ADDITIONAL DRIVERS
   The "additional drivers" flow shows an information message but does not
   allow entry of additional driver details. This feature is stubbed out with
   a notice and must be built out for production.

6. ACCESSIBILITY (WCAG 2.1 AA)
   The app uses semantic HTML, labelled form controls, and focus styles.
   A full WCAG 2.1 AA audit has not been completed. Automated testing with
   axe-core or similar should be performed before production release.

7. NO REAL-WORLD INSURER PANEL
   The "panel of car insurers" referenced in the UI is fictional for this demo.
   Integration with an actual aggregator or insurer panel API is required for
   live operation.

8. FAVICON NOT INCLUDED
   No favicon.ico file has been created. A Swinton Go! branded favicon should
   be added before production deployment.

9. COOKIE / CONSENT BANNER
   No cookie consent banner is present. Under UK PECR / UK GDPR rules, a
   consent mechanism is required before setting any non-essential cookies.
   Note: this demo only uses sessionStorage (not cookies), but a production
   version with analytics or tracking would require a consent banner.

10. GOOGLE FONTS CDN DEPENDENCY
    The stylesheet imports Inter from Google Fonts via CDN. If users are offline
    or the CDN is unavailable, the app falls back to system sans-serif fonts.
    For production, consider self-hosting the font files.

11. REGULATORY / LEGAL COPY
    Company registration details, FCA reference numbers, and registered office
    address have been removed as they are specific to Swinton Go! / BiSL Limited
    and must be confirmed with the legal / compliance team before going live.

12. IE / LEGACY BROWSER SUPPORT
    The app uses modern CSS custom properties, CSS Grid, and ES6+ JavaScript.
    Internet Explorer is not supported. Minimum supported browsers are the
    latest two versions of Chrome, Firefox, Safari, and Edge.
